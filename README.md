A robust Python-based automation utility designed to de-clutter workspace directories by categorizing files, organizing them into a date-based directory hierarchy, and generating a comprehensive CSV movement report for audit purposes.

## Key Features

* **Dynamic Classification:** Automatically maps files to predefined operational categories (e.g., *Reports*, *Financial Statements*) based on file extensions.
* **Chronological Directory Architecture:** Dynamically generates nested subfolders following a `Category/Year/Month` schema using OS-level file metadata.
* **Collision Handling & Versioning:** Implements an iterative `while` loop logic to prevent data loss. If a filename collision occurs at the destination, the script auto-increments the filename (e.g., `document_1.pdf`).
* **Atomic Reporting:** Produces a `report_movements.csv` log using absolute paths, ensuring 100% traceability of the migration process.
* **Cross-Platform Compatibility:** Engineered with `os.path` abstraction to ensure seamless execution across Windows, macOS, and Linux environments.

## Logic & Execution Pipeline

The script follows a structured execution flow to ensure data integrity:

1.  **Environment Validation:** Checks for source directory existence and initializes target paths using `os.makedirs`. This ensures the script is "zero-config" and ready to run immediately.
2.  **Mapping Optimization:** Performs a "reverse index" on the category dictionary to create a high-performance $O(1)$ lookup table for file extensions.
3.  **Metadata Extraction:** Retrieves creation/modification timestamps to define the target historical path.
4.  **Transaction & Logging:** Executes file migration via `shutil.move` within a `try-except` block. The CSV log is updated **only** upon a successful I/O operation.

## Installation & Usage

1.  **Clone the Repository:**
    ```bash
    git clone [https://github.com/your-username/python-file-organizer.git](https://github.com/your-username/python-file-organizer.git)
    cd python-file-organizer
    ```

2.  **Run the Script:**
    Launch the script to automatically generate the necessary directory structure:
    ```bash
    python app.py
    ```

3.  **Input Data:**
    Place the files to be sorted into the newly created `FileToSort` directory.

4.  **Review Output:**
    Sorted files will be located in `OutputData`. The execution log is available in `report_movements.csv`.
